# Security Policy

## Supported Versions

The current main branch is maintained for source-code review, documentation updates, and reproducibility improvements.

## Reporting a Vulnerability

If you find a vulnerability or a data exposure risk:

1. Do not publish private details in a public issue.
2. Open a minimal report with the affected file and a short description.
3. Avoid attaching private datasets, images, model binaries, or device-specific configuration.
4. Provide a safe reproduction case using placeholder paths when possible.

The maintainers will review reports and publish fixes when appropriate.

## Data Handling

This repository should not contain:

- private datasets
- private PCB images
- generated inspection images
- model binaries
- machine-specific paths
- board network addresses
- credentials

Please review files before submitting changes.
